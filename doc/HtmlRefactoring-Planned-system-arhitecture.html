HTML Refactoring - Planned System Arhitecture

This program plays a role of a HTML/CSS parser/translator. Magazine articles are formated by graphic designers to looks good on a paper. It is done by Adobe InDesign program which then export coresponding HTML & CSS file for the each article.

When InDesign exports HTML & CSS files they looks bad and we want all our articles not only to looks good but also to have standardized appearance on a web page. That is the duty of the HTML Refactoring program.

CSS Files

CSS files are loaded and parsed before I can parse HTML files.

In this system, CSS rules found in CSS files are divided into two categories:

  • Vendor (known) rules
  • Generic (unknown) rules

Vendor (known) rules

Vendor rules are predefined set of classes used by graphic editors of a magazine which on a functional level describe part of a text in an article. These source CSS rules are directly translated to destination CSS rules based on coresponding translation table.

Translation table is defined by the Scripts

Since there is no need to interpret properties of Vendor rules they are ignored when processing CSS files.

Generic (unknown) rules

When source HTML files of an article is generated by the InDesign it uses Vendor rules for all known styles used by graphic editors. For everything else InDesign creates it's own generic CSS rules which have to be interpreted, simplified and applied to destination HTML file.

CssFile

CssFile is generic collection of css rules.

CssRule

CssRule consists of selector part and property part

While we read CSS file we can encounter some selectors more then once, so we have to take care whether we should create new CSS rule or we should ammend existing one.

CssSelector

CssSelector can be written as a single selector or as a list of many selectors separated by commas and white-space characters.

Since CSS rules can be amended we will add separate CSS rule for each selector from the selectors list.

CssProperty

CssProperty is a semi-colon separated list of name-value pairs of a different CSS properties.

Since the goal of our translation is to simplify and standardize the output files we are going to ignore some properties, while we have to interpret and adjust the others.
